# Модуль учета движения товара

Демонстрационный модуль для курсового проекта по теме **«Проектирование и разработка информационной системы для складского учета»**.

## Назначение
Модуль автоматизирует учет движения товара в системе складского учета и поддерживает основные бизнес-операции:
- регистрацию товаров и складов;
- приход товара на склад;
- расход товара со склада;
- перемещение товара между складами;
- корректировку остатков по результатам инвентаризации;
- формирование отчета по остаткам и списку товаров с критическим уровнем запаса.

## Требования
- Python 3.11+
- pip

## Установка зависимостей
```bash
python -m venv .venv
source .venv/bin/activate        # Linux/macOS
# .venv\Scripts\activate         # Windows
pip install -r requirements.txt
```

## Запуск приложения


## Примеры запросов
### 1. Создание товара
```bash
curl -X POST http://127.0.0.1:8000/products \
  -H "Content-Type: application/json" \
  -d '{
    "sku": "SKU-100",
    "name": "Сканер штрихкодов",
    "unit": "шт",
    "min_stock": 3
  }'
```

### 2. Создание склада
```bash
curl -X POST http://127.0.0.1:8000/warehouses \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Основной склад",
    "code": "MAIN-01"
  }'
```

### 3. Приход товара
```bash
curl -X POST http://127.0.0.1:8000/movements/receipt \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": "ID_ТОВАРА",
    "warehouse_id": "ID_СКЛАДА",
    "quantity": 10,
    "operator": "Иванов И.И.",
    "document_number": "PR-2026-001"
  }'
```

### 4. Просмотр остатков
```bash
curl http://127.0.0.1:8000/stock
```

## Описание ключевых файлов
- `models.py` — доменные сущности: товар, склад, документ движения.
- `repository.py` — уровень доступа к данным. Для курсового проекта используется хранение в памяти.
- `service.py` — бизнес-логика модуля.
- `api.py` — REST API для взаимодействия с модулем.
- `test_service.py` — unit-тесты бизнес-логики.
